# v2 最终目标

本项目最终目标是做一个“宇智能”：AI 获客与内容运营客户端。

## 项目正式名称

项目正式命名为“宇智能”。定位为面向中小企业的 AI 获客与内容运营客户端。

## Phase 2B 产品聚焦

Phase 2B 起，宇智能第一条可演示产品线聚焦为“面向东莞本地制造业企业的 AI 获客与内容运营工作台”。

普通制造业用户看到的是制造业获客路径，而不是通用 Agent 平台：

1. 我要找客户。
2. 我要生成宣传内容。
3. 我要处理评论/私信。
4. 我要养账号。
5. 我要把客户转给工厂。
6. 今日待办。
7. 获客报告。

高级模式仍保留 OpenClaw / LobsterAI / Skill / MCP 的底层扩展能力，但默认不暴露给普通客户。

## 目标形态

用户下载安装 Windows 客户端后，可以：

1. 登录会员账号。
2. 输入自己的产品、行业、城市、目标客户。
3. 系统自动做公开信息研究。
4. 系统识别潜在客户和需求信号。
5. 系统给客户评分。
6. 系统生成小红书、抖音、公众号等内容建议。
7. 系统生成评论、私信、跟进草稿。
8. 所有外部动作进入审批或草稿。
9. 用户决定是否执行。
10. 系统记录结果并持续复盘。

## 第一版闭环

用户安装客户端
→ 登录会员
→ 输入推广目标
→ 系统生成商机判断
→ 系统生成目标客户类型
→ 系统生成内容建议
→ 系统生成待办
→ 用户人工执行
→ 用户回填结果
→ 系统复盘

## 第一版不做

- 自动发帖
- 自动评论
- 自动私信
- 自动邮件
- 绕过平台限制
- 抓取登录后敏感数据
- 购买非法数据
- 完整 CRM
- 完整支付系统
- 大而全控制台

## 后续高级能力

后续高级能力可以做，但必须：

- 默认关闭
- 用户手动开启
- 开启前提示风险
- 开启后记录审计日志
- 支持随时关闭
- 支持限流
- 支持回滚

## 组件路线

优先验证：

1. LobsterAI：桌面客户端壳。
2. Sim.ai：Agent 工作流编排。
3. domestic_signal_growth Skill：国内营销信号穿透。
4. SearXNG / Crawl4AI / Firecrawl / Meilisearch：搜索和抓取。
5. PocketBase：会员授权。
6. Twenty / Chatwoot / n8n：后续 CRM、客服和自动化。

## 成功标准

产品成功不是因为 Agent 图好看，而是因为：

1. 能找到客户。
2. 能判断客户需求。
3. 能生成可用内容。
4. 能形成待办。
5. 能促进人工跟进。
6. 能持续复盘。
7. 能让客户愿意续费。
