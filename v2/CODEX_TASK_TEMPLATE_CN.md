# Codex 任务入口模板

每次给 Codex 新任务时，必须包含以下结构。

## 任务名称

填写任务名称。

## 背景

说明为什么做这个任务。

## 目标

说明本次只完成什么。

## 非目标

明确本次不做什么。

## 最高原则

本任务必须遵循：

- 开源优先
- 最小改动
- 不从零写完整系统
- 不真实外发
- 不读取密钥
- 高敏感功能默认关闭但允许用户后续手动开启

## 必须调用的子 Agent

默认必须调用：

- open_source_scout
- product_fit_reviewer
- integration_engineer
- safety_reviewer
- test_engineer
- report_writer

如涉及 UI，再调用：

- ux_reviewer

## 开源优先检查

必须回答：

1. 有哪些成熟开源项目？
2. 哪个最适合？
3. 是否能直接部署？
4. 是否能通过 API/Webhook/CLI/MCP/Skill 接入？
5. 为什么不从零写？

## 最小改动计划

必须回答：

1. 修改哪些文件？
2. 不修改哪些文件？
3. 是否影响旧系统？
4. 如何回滚？

## 测试要求

必须列出具体测试命令和预期结果。

## 交付要求

必须输出：

1. 修改文件清单
2. 测试结果
3. 安全检查结果
4. 中文报告路径
5. commit hash
6. 下一步建议
