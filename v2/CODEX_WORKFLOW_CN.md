# Codex 工作流

## 目标

Codex 是本项目的开发执行者，不是业务运行 Agent。

Codex 负责：

- 开源项目选型
- 部署验证
- 集成适配
- 最小补丁
- 测试
- 报告
- 回滚建议

Codex 不负责：

- 从零写完整系统
- 自行决定产品主线
- 绕过平台限制
- 真实外发
- 自动使用密钥

## 每个实质开发任务必须先读取

- `D:\OpenClaw\AGENTS.md`
- `D:\OpenClaw\v2\PHASE_GATE_CHECKLIST_CN.md`
- `D:\OpenClaw\v2\V2_FINAL_GOAL_CN.md`

## 标准开发流程

每个实质开发任务必须按以下流程执行：

1. 运行 `open_source_scout`，判断是否已有成熟开源项目、官方 SDK/API、插件/Webhook/CLI/MCP/Skill 接入方式。
2. 运行 `product_fit_reviewer`，判断是否符合 AI 获客与内容运营客户端主线。
3. 运行 `integration_engineer`，制定最小集成方案。
4. 运行 `safety_reviewer`，检查高敏感能力、外部动作、账号、密钥、导出和平台限制。
5. 运行 `test_engineer`，给出测试命令、预期结果、实际结果和阻塞判断。
6. 运行 `report_writer`，输出中文阶段报告。
7. 如涉及 UI，再运行 `ux_reviewer`。

## 子 Agent 调用规则

### 必须调用 open_source_scout 的情况

- 新增功能
- 新增组件
- 新增服务端能力
- 新增客户端能力
- 新增工作流
- 新增数据源
- 新增社媒能力
- 新增会员能力

### 必须调用 product_fit_reviewer 的情况

- 涉及普通用户使用方式
- 涉及产品入口
- 涉及商用可行性
- 涉及是否继续投入某组件

### 必须调用 integration_engineer 的情况

- 部署开源项目
- 接 API
- 接 Webhook
- 接 Skill
- 接 MCP
- 接本地脚本
- 做桥接服务

### 必须调用 safety_reviewer 的情况

- 涉及账号
- 涉及密钥
- 涉及登录态
- 涉及发帖
- 涉及评论
- 涉及私信
- 涉及邮件
- 涉及导出
- 涉及删除
- 涉及支付
- 涉及真实外部平台

### 必须调用 test_engineer 的情况

- 有代码变更
- 有脚本变更
- 有依赖安装
- 有服务启动
- 有接口调用
- 有打包
- 有数据库变更

### 必须调用 report_writer 的情况

- 每一个阶段任务结束
- 每一个 PoC 结束
- 每一次失败需要决策
- 每一次提交前

## 阶段门禁

任何阶段提交前必须回答：

1. 是否优先检查过开源方案？
2. 是否存在可直接部署项目？
3. 是否存在官方 SDK/API？
4. 是否真的需要改源码？
5. 改动范围是否最小？
6. 是否涉及高敏感功能？
7. 高敏感功能是否默认关闭？
8. 是否有测试结果？
9. 是否有中文报告？
10. 是否有回滚方式？

如果以上任一项无法回答，不允许提交。
